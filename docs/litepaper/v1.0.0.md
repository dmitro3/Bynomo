# Bynomo Litepaper (v1.0.0)

Last updated: 2026-04-01

## 1. Abstract

Bynomo is an on-chain binary options protocol focused on short-duration prediction rounds across crypto, forex, metals, and stocks. The protocol combines oracle-backed settlement, multi-chain wallet support, and a risk-controlled treasury model to provide transparent market participation with reduced trust assumptions.

## 2. Problem Statement

Most prediction products are either opaque, custody-heavy, or fragmented across chains and assets. Users face poor settlement transparency, delayed payouts, and unclear risk accounting. Bynomo addresses these gaps with deterministic round logic, visible payout flows, and unified UX across supported networks.

## 3. Product Overview

Users connect a wallet, select market and duration, choose direction, and place a position. At round close, the protocol resolves using oracle-attested prices and settles wins/losses according to predefined payout rules. Public dashboards provide protocol health, trading activity, and leaderboard visibility.

## 4. Protocol Architecture

Bynomo uses a Next.js application layer, server-side execution routes, Supabase-backed persistence, and oracle market data ingestion. Administrative operations are server-authenticated, and sensitive database access is restricted with RLS. Settlement-critical actions are routed through signed, auditable backend flows.

## 5. Security Model

The protocol applies defense-in-depth: strict admin auth, short-lived admin sessions, CSP and security headers, restricted API CORS, and deny-by-default Supabase RLS. Client-side trust is minimized by moving critical updates (such as referral accounting) to server-controlled endpoints.

## 6. Treasury & Risk Controls

Bynomo manages treasury exposure through configurable payout logic, withdrawal review workflows, and chain-specific accounting. Operational controls include status flags for user balances, ban/freeze tooling, and full audit trails for balance mutations and withdrawal decisions.

## 7. BYNOMO Token Utility

The BYNOMO token is designed for ecosystem utility, including campaign incentives, potential fee incentives, and partner integrations. Any future emissions, staking, or governance mechanics are introduced through versioned updates to this litepaper with clear migration notices.

## 8. Roadmap

Near-term milestones include expanded market coverage, improved execution telemetry, refined anti-abuse controls, and advanced user analytics. Mid-term priorities include deeper chain integrations, improved liquidity partnerships, and formal external security review cycles.
